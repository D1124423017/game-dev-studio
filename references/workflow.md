# 工作流程參考

## 新遊戲啟動

1. 接收使用者想法
2. 切換製作人角度：釐清範圍、用途、時間、MVP
3. 切換企劃角度：玩法、核心循環、爽點
4. 切換技術角度：引擎、平台、架構
5. 切換美術角度：風格、資產需求
6. 切換 UI/UX 角度：必要畫面、操作流程
7. 切換 UI Motion 角度：HUD、選單、卡牌、獎勵、Combo、結果畫面的動態回饋
8. 整合產出：方向提案 + 推薦方向 + 風險 + 下一步

## 開發階段

1. 先確認影響範圍
2. 先讀現有程式碼再改
3. 如果涉及 UI / HUD / 玩家回饋，先確認動效方案與工具選型
4. 實作後驗證
5. 回報變更與風險

## 完成階段

1. 犀利測試玩家發表意見
2. 檢查視覺、UI motion 與音效完整性
3. 檢查 reduced motion、效能、可讀性與輸入延遲
4. 檢查文件與驗收標準
5. 回報最終產出

## UI Motion / Game Feel Review

- Motion should be reviewed after static UI/UX and before final polish.
- 視覺相關任務除了圖片資產，也要檢查互動動效。
- UI motion should improve clarity, feedback, rhythm, and game feel, not add decorative noise.
